North Star: Build Systems That Matter

90-Day Launch Play by Play

Use this exactly as written to move from room-build to field-ready prototype.

Weeks 1-4: Build the Core Stack
  • Week 1: device firmware baseline on Pico and ESP32, including logging and reconnect.
  • Week 2: Docker services for API, MQTT broker, and Postgres persistence.
  • Week 3: command-and-control path with role-based API keys and audit logs.
  • Week 4: dashboard showing device status, uptime, and event stream.
Weeks 5-8: Build a Vertical Pilot
  • Week 5: choose pilot narrative, hotel announcements, generator monitoring, or fire-event bridge.
  • Week 6: implement scenario-specific workflows and role screens.
  • Week 7: run resilience test, outages, bad payloads, stale data, and clock drift.
  • Week 8: produce deployment guide and demo recording.
Weeks 9-12: Prepare for Real Use
  • Week 9: add access controls and event retention policy.
  • Week 10: create operational runbooks for incidents and maintenance.
  • Week 11: run user acceptance with one decision-maker.
  • Week 12: convert to paid pilot or support contract.
Rule: every 90-day cycle must end with one deployment-ready package and one signed commercial next step.

Hotel Blueprint: Replace Paper Announcements

Target scenario: hotel operations can publish announcements to in-room TV and phone channels in minutes.

System Design

  • Operator creates message by language, floor, room class, and urgency.
  • Backend pushes content to TV middleware and mobile notification providers.
  • Room-level delivery acknowledgments are logged in Postgres.
  • Escalation path triggers if delivery falls below threshold.

Pilot Scope

  • Start with 1 floor and two message types: service announcement and safety advisory.
  • Success metric: 95 percent delivery in 3 minutes.
  • Export compliance report to replace manual paper distribution logs.
Respect safety, privacy, and hotel policy requirements. Security and auditability are part of the product.
